SRC Testing

Why test SRC components?

  • Manual testing on SRC does not scale
    • Different OS’s/versions
    • Slow deploy times
  • We only find out that something is broken when it’s too late

SRC testing challenges

  • Dependencies on base image
  • Dependencies on required SURF components
    • SRC-OS
    • SRC-CO
  • Ansible-in-Ansible
    • SRC-External

Solution

  • Packer
    • Base images for each OS flavour
    • Execute required components
    • Store these images on ghcr.io
  • Molecule
    • Deploy our playbooks and roles on those images
    • Run additional tests
  • GitHub Actions
    • Run molecule tests in CI

Molecule

default configuration for SRC testing:

  • create.yml (provided by Molecule): use Docker/Podman to start a container using one of our base images
  • prepare.yml: copy the required component onto the workspace
  • converge.yml: execute the component on the workspace
    • Same method as SRC External
    • Set component parameters
  • each scenario: molecule.yml

Molecule

…or with PyTest/testinfra:

def test_passwd_file(host):
    passwd = host.file("/etc/passwd")
    assert passwd.contains("root")
    assert passwd.user == "root"
    assert passwd.group == "root"
    assert passwd.mode == 0o644


def test_nginx_is_installed(host):
    nginx = host.package("nginx")
    assert nginx.is_installed
    assert nginx.version.startswith("1.2")


def test_nginx_running_and_enabled(host):
    nginx = host.service("nginx")
    assert nginx.is_running
    assert nginx.is_enabled

Molecule

To tell molecule to use our custom SRC configuration:

molecule -c molecule/default/molecule.yml -s <scenario-name> test

Molecule

Don’t destroy the container after tests are done:

molecule -c molecule/default/molecule.yml -s <scenario-name> test --destroy=never

…now login to the container!

(Or just run the converge step.)
